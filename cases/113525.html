Hi Winder,

Regarding PON reset, according to engineering, it is structurally impossible to reset only a PON port to its default settings with a single command.

Regarding ONU deactivation, using "no onu-profile ONU_ID" will result in the ONU no longer receiving service, while using "no onu ONU_ID" will exclude the ONU from all settings on the PON port, even if it remains physically active.
