Resolución

Por parte de Zhone informaron lo siguiente :

Based on your test topology, the MTU size needs to be adjusted not only on the ONT but also on the uplink and intergroup interfaces to ensure proper operation.

Below is a sample config to set the MTU to 9230 when an ONT is attached to gpon0/13 and the uplink interface is eth0/17.

traffic-profile V2274T

tcont 1

gemport 1/1

dba-profile GPON

mapper 1

gemport count 1

bridge 1

ani mapper 1

uni eth 1

extended-vlan-tagging-operation V2274T

max-frame 9230

apply

!

interface intergroup 0/7

mtu 9230

!

interface ethernet 0/17

interface-usage network-port

speed 1g

bridge-group 1 spanning-tree disable

switchport mode trunk

switchport trunk allowed vlan add 33

vlan ingress mapping s-vid 33 swap-s-vid 33

mtu 9230

service-policy type queuing output ethernet 0/17.policy-map.queuing.def

no shutdown

ip dhcp snooping trust

ipv6 dhcp snooping trust

!

Note: Each intergroup consists of two XGS-PON and two GPON ports. You can verify intergroup members using the following command:
