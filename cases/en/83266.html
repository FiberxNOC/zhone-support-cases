Case content

Summary

When attempting to provision 3 ONUs, the equipment model was not displayed. The model was manually inputted but the configuration was not adopted. The bridge created did not appear and the ONU still appears in the show. A Zhone configuration fix was received to resolve the issue.

Problem detail

Upon attempting to provision 3 ONUs, the Equipment Model was not visible, so it was manually set but the configuration was not adopted. The bridge created did not appear and the ONU continues to show up in the show commands. Configuration from Zhone was received to resolve the issue.

Resolution

Check the existing setting of the card-init-string field by using the card show command.

If the setting is null or 32onu, then proceed to Step 2.

zSH> card show

Card Init String: --------> {}

Or,

zSH> card show

Card Init String: --------> {32onu}

2. Set the default ONU size to 64onu by changing the

card-init-string to 64onu.

zSH> update card-profile card-atm-configuration = notapplicable 1/1/10550

zSH> update card-profile card-init-string = 64onu 1/1/10550

zSH> card show

Card Init String: --------> {64onu}
